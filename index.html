Critical · CVE-2025-4471
Heap Overflow to Full Kernel Control via io_uring Async Cancel Path

A reliable privilege escalation chain from an unprivileged user to ring-0, exploiting a race window in Linux 6.7.x's io_uring cancellation handler. We trace the off-by-one through four exploitation stages with a complete weaponized PoC.

heap overflow LPE kernel io_uring Linux 6.7.x
Linux Kernel · June 2025
Status: Patched (v6.8)
CVSS 9.8

The root cause is a single off-by-one in io_async_cancel() that leaves a request structure partially freed. On SMP systems this race window is deterministic and reliable within seconds.

// about zerosploit

We find what
others missed.

ZeroSploit is an independent vulnerability research group. We select targets based on real-world deployment — software that organizations depend on — and audit until we find what the vendor missed.

Every finding becomes a complete technical writeup. How we found it, how we exploited it, what defenders can do. No teaser posts. No paywalled details. No marketing.

We believe defenders and attackers should not have asymmetric access to the same information. So we publish everything.
